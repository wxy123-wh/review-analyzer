## ADDED Requirements

### Requirement: 模块必须与评论决策主线相关
系统 MUST 确保项目经理可见模块都能解释其与评论分析、问题识别、改进动作或验证闭环的关系。

#### Scenario: 导航模块具备业务关联说明
- **WHEN** 用户查看导航模块
- **THEN** 每个模块都有明确的评论业务关联说明或上下文提示

### Requirement: 动作模块语义业务化
系统 MUST 将“动作”模块明确表达为“改进动作”，并展示动作与问题的绑定关系。

#### Scenario: 动作与问题绑定可见
- **WHEN** 用户查看改进动作列表
- **THEN** 页面可看到每个动作关联的问题、状态与时间信息

### Requirement: 混沌演练模块治理
系统 MUST 对“混沌演练”模块采用收敛策略，避免与评论场景脱节。

#### Scenario: 默认隐藏弱相关演练
- **WHEN** 系统运行在项目经理演示模式
- **THEN** 与评论主线弱相关的混沌演练模块不在默认导航中展示

#### Scenario: 保留演练时需评论链路映射
- **WHEN** 系统配置为显示演练模块
- **THEN** 页面必须展示“该演练如何影响评论同步/分析/看板更新”的映射说明
