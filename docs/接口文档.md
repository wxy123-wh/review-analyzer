# 接口文档

本文只记录当前代码中已经实现的接口。未在代码中确认的行为，一律不写；无法从当前代码确认的内容，会明确标注。

## 1. backend 接口

### 1.1 GET /api/v1/health
- 作用：返回后端健康状态。
- 请求参数：无。
- 返回字段：
  - `status: string`
  - `timestamp: Instant`
- 默认行为：固定返回 `status = "UP"`。
- 异常处理：代码中未定义业务异常分支。

### 1.2 POST /api/v1/sync/start
- 作用：创建同步任务；当 `provider=onebound` 时立即尝试调用 OneBound。
- 请求体：
  - `provider: string`，必填，非空
  - `platform: string`，可选，为空时后端归一化为 `taobao`
  - `targetProductCode: string`，必填，非空
- 返回字段：
  - `jobId: string`
  - `provider: string`
  - `platform: string`
  - `targetProductCode: string`
  - `status: string`
  - `startedAt: Instant`
  - `fetchedCount: int`
  - `errorMessage: string | null`
- HTTP 状态：`202 Accepted`
- 默认行为：
  - 非 `onebound` provider：返回并持久化 `QUEUED`
  - `onebound`：立即执行一次外部请求并返回 `SUCCEEDED` 或 `FAILED`
- 边界与异常：
  - `ONEBOUND_API_KEY` 缺失会失败。
  - `ONEBOUND_API_SECRET` 缺失时会回退使用 key。
  - 错误消息会打码 `key=` 与 `secret=`。

### 1.3 GET /api/v1/sync/jobs/{id}
- 作用：查询同步任务。
- 路径参数：`id`
- 成功返回：同 `SyncJobResponse`
- 失败返回：`404`，JSON 结构：
  - `message: "sync job not found"`
  - `jobId: string`
- 边界条件：ID 非数字时同样查询不到。

### 1.4 POST /api/v1/analysis/start
- 作用：创建分析任务。
- 请求体：
  - `productCode: string`，必填，非空
- 返回字段：
  - `jobId: string`
  - `productCode: string`
  - `status: string`
  - `startedAt: Instant`
- HTTP 状态：`202 Accepted`
- 默认行为：当前代码只创建 `QUEUED` 任务，不执行真实分析。

### 1.5 GET /api/v1/analysis/jobs/{id}
- 作用：查询分析任务。
- 路径参数：`id`
- 成功返回：同 `AnalysisJobResponse`
- 失败返回：`404`，JSON 结构：
  - `message: "analysis job not found"`
  - `jobId: string`

### 1.6 POST /api/v1/demo-data/init
- 作用：初始化或刷新演示评论数据。
- 请求体：可为空；若传则结构为：
  - `productCode?: string`
- 返回字段：
  - `seedKey: string`
  - `productCode: string`
  - `dataVersion: string`
  - `targetReviewCount: int`
  - `insertedReviewCount: int`
  - `updatedReviewCount: int`
  - `totalReviewCount: int`
  - `durationMs: long`
  - `initializedAt: Instant`
- 默认行为：
  - 请求体缺失或 `productCode` 空白时，回退到 `demo-earphone`
  - 目标评论数固定为 `100`
  - 重复初始化时走 upsert，不会无限新增

### 1.7 GET /api/v1/issues
- 作用：返回问题优先级列表。
- 查询参数：
  - `productCode?: string`，默认 `demo-earphone`
- 返回字段：
  - `items: IssueItem[]`
- `IssueItem` 字段：
  - `issueId: string`
  - `title: string`
  - `aspect: string`
  - `priorityScore: number`
  - `evidenceSummary: string`
- 默认行为：
  - 数据不足时返回兜底问题项，不是 HTTP 错误。
  - 结果按 `priorityScore` 降序排列。

### 1.8 GET /api/v1/compare
- 作用：返回竞品对比分数。
- 查询参数：
  - `productCode?: string`，默认 `demo-earphone`
- 返回字段：
  - `productCode: string`
  - `items: CompareItem[]`
- `CompareItem` 字段：
  - `aspect: string`
  - `ourScore: number`
  - `competitorScore: number`
  - `gap: number`
- 默认行为：当前代码直接返回静态数据，不依赖数据库。

### 1.9 GET /api/v1/trends
- 作用：返回趋势点。
- 查询参数：
  - `productCode?: string`，默认 `demo-earphone`
  - `aspect?: string`，默认 `battery`
- 返回字段：
  - `productCode: string`
  - `aspect: string`
  - `points: TrendPoint[]`
  - `notice: string | null`
- `TrendPoint` 字段：
  - `period: string`
  - `negativeRate: number`
  - `mentionVolume: number`
- 默认行为：按 ISO 周聚合。
- 边界与异常：
  - 无数据时 `points=[]`，并通过 `notice` 给提示。
  - 聚合失败时也返回空数组与提示，而不是抛 5xx 业务包装。

### 1.10 GET /api/v1/wordcloud
- 作用：返回词云数据。
- 查询参数：
  - `productCode?: string`，默认 `demo-earphone`
  - `aspect?: string`，默认 `all`
- 返回字段：
  - `productCode: string`
  - `aspect: string`
  - `items: WordCloudItem[]`
  - `notice: string | null`
- `WordCloudItem` 字段：
  - `keyword: string`
  - `frequency: number`
  - `weight: number`
  - `sentimentTag: string`
- 默认行为：最多返回 24 个关键词。
- 边界与异常：
  - 无数据时 `items=[]`。
  - 仅支持代码里定义的停用词与词元规则。

### 1.11 POST /api/v1/actions
- 作用：登记改进行动。
- 请求体：
  - `productCode: string`，必填，非空
  - `issueId: string`，必填，非空
  - `actionName: string`，必填，非空
  - `actionDesc?: string`
- 返回字段：
  - `actionId: string`
  - `productCode: string`
  - `issueId: string`
  - `actionName: string`
  - `actionDesc?: string`
  - `status: string`
  - `createdAt: Instant`
- HTTP 状态：`201 Created`
- 默认行为：状态固定为 `PLANNED`。
- 隐式逻辑：如果 `productCode` 对应产品不存在，会自动创建产品，且 `brand=unknown`。

### 1.12 GET /api/v1/validation
- 作用：查询动作验证结果。
- 查询参数：
  - `actionId?: string`
- 返回字段：
  - `items: ValidationItem[]`
  - `notice: string | null`
- `ValidationItem` 字段：
  - `actionId: string`
  - `beforeNegativeRate: number`
  - `afterNegativeRate: number`
  - `improvementRate: number`
  - `summary: string`
- 默认行为：
  - 传 `actionId`：只查单个动作
  - 不传：查全部动作
- 边界与异常：
  - 动作不存在时 `items=[]`，`notice` 提示未找到
  - 完全没有动作时 `items=[]`，`notice` 提示暂无改进动作
  - 样本不足时 summary 会明确说明无法形成稳定结论

## 2. nlp-service 接口

### 2.1 GET /health
- 作用：返回 NLP 服务健康状态。
- 返回字段：
  - `status: string`
  - `timestamp: string`
- 默认行为：固定返回 `status = "UP"`。

### 2.2 POST /analyze
- 作用：执行规则型评论分析。
- 请求体：
  - `jobId: string`
  - `productCode: string`
  - `reviews: string[]`
- 返回字段：
  - `jobId: string`
  - `aspectSentiments: AspectSentiment[]`
  - `issueClusters: IssueCluster[]`
- `AspectSentiment` 字段：
  - `reviewIndex: int`
  - `aspect: string`
  - `polarity: string`
  - `score: float`
  - `confidence: float`
- `IssueCluster` 字段：
  - `aspect: string`
  - `title: string`
  - `mentionCount: int`
- 默认行为：
  - 方面识别完全基于关键词匹配
  - 情感识别完全基于关键词匹配
  - 只有负向方面会进入 `issueClusters`
- 边界条件：
  - 未命中任何方面关键词时返回 `general`
  - 未命中正负向词时返回 `NEUTRAL`

## 3. 前端侧调用约定
- `frontend/src/api/client.ts` 为前端唯一集中接口封装。
- 默认后端基地址：`VITE_API_BASE_URL`；缺失时用 `http://localhost:8080`。
- 请求超时：`10000ms`。
- 超时判定：`ECONNABORTED`、HTTP `408`、HTTP `504`。
- 前端测试模式下，大量调用直接使用本地 mock 返回，不访问真实服务。
