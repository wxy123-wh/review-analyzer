# 前端文案审校清单

适用范围：`frontend/src`（Vue 组件与前端文案数据）、`README.md`。

## 1. 术语一致性

- [ ] 全站对外文案统一使用“演示数据”，不混用“占位数据/示例数据/demo 数据”。
- [ ] 模块名称与导航名称保持一致（总览、问题、对比、趋势、动作、验证、流水线、智能体、可解释性、混沌演练、报告中心）。
- [ ] 状态词统一：`UP/DOWN/UNKNOWN` 在界面层分别展示为“正常/异常/未知”。

## 2. 禁用词检查

- [ ] 禁用词不出现在前端可见文案与 README：`炫技`、`PLACEHOLDER`、`Quantum`、`Operator`。
- [ ] 本地执行 `npm --prefix frontend run check:copy` 结果通过。
- [ ] 若新增文案文件，确认已纳入禁用词扫描范围。

## 3. 中文表达质量

- [ ] 中文优先，避免中英文混杂导致语义断裂。
- [ ] 同一概念在不同页面使用同一叫法（例如“报告预览”不出现“report preview”等替代表达）。
- [ ] 提示语明确可执行（例如失败提示、登录提示、空态提示）。

## 4. README 项目经理视角

- [ ] README 包含中文“项目经理说明”，覆盖目标、交付、价值三项。
- [ ] README 中演示流程步骤与前端实际入口文案一致。
- [ ] README 中接口说明与界面术语一致（统一“演示数据”）。

## 5. 回归确认

- [ ] 执行 `npm --prefix frontend test` 通过。
- [ ] 核验关键路径：登录 -> 总览 -> 各演示模块 -> 报告中心 -> 验证。
- [ ] 若发现术语偏差，先修正文案，再补测试断言与审校清单记录。
